# 004 — 정답/오답 피드백 UI 개선

## 목적

추리 결과(정답/오답)가 나왔을 때 시각적 피드백이 약해서 사용자가 인지하기 어렵다.
또한 계속 추리 / 턴 종료 버튼이 너무 작아서 찾기 힘들다.
이 작업에서 결과 피드백을 눈에 띄게 개선하고 액션 버튼을 명확하게 키운다.

## 범위

- 변경 파일: `client/src/pages/game/GuessModal.tsx` (또는 CenterZone 내 결과 표시 영역)
- 변경 없는 파일: 서버 로직, 소켓 이벤트, 타입 — 모두 그대로

## 작업 내용

### 1. 결과 오버레이 (정답/오답 전체화면 or 모달)

- 추리 결과가 들어오면 `guess_result` 이벤트 기반으로 결과 화면을 크게 표시
- **정답**: 초록색 배경 + ✓ 아이콘 + "정답!" 텍스트 + 공개된 타일 표시
- **오답**: 빨간색 배경 + ✗ 아이콘 + "오답" 텍스트 + 패널티 타일 표시

### 2. 액션 버튼 개선

- 정답 후 "계속 추리" / "턴 넘기기(skip)" 버튼을 크고 명확하게
- 버튼 최소 높이 48px 이상, 명확한 레이블
- 오답 후에는 버튼 없이 자동으로 닫히거나 "확인" 하나만

### 3. 애니메이션

- 결과 표시 시 fade-in 또는 scale 등장 애니메이션 추가 (Tailwind `animate-` 클래스 활용)

## 구현 순서

1. `GuessModal.tsx` 또는 `CenterZone.tsx`에서 `guessResult` 상태 처리 부분 확인
2. 결과 오버레이 컴포넌트 / 섹션 UI 교체
3. 버튼 크기 및 레이블 개선
4. 브라우저에서 정답/오답 시나리오 직접 테스트
