# 002 — 패키지 구조: pnpm 모노레포

## 결정
단일 레포에 `client/`, `server/` 분리, 패키지 매니저는 pnpm.

## 선택지
- 레포 분리 (client repo + server repo)
- 모노레포 (npm) 
- 모노레포 (pnpm) ← 선택

## 이유
- 클라이언트/서버 타입을 공유할 가능성이 있어 단일 레포가 유리
- pnpm은 npm 대비 설치 속도 빠르고 디스크 효율적
- 레포 분리 시 소켓 이벤트 인터페이스 동기화가 번거로움

## 트레이드오프
- pnpm 워크스페이스를 쓰지 않아 client/server 간 코드 직접 공유는 불가
- 빌드/배포를 각각 따로 설정해야 함 (Vercel + Render 분리)
